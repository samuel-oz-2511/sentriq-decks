Every application is submitted as text.
Resume. Cover letter. Form answers. All text. Always.
AI can generate any text in under a second.
And apply to 10,000 jobs in an afternoon. Automatically.
The ATS was not built for this world.

What the ATS sees
What was submitted.
Not how it was created.
The ATS stores the resume. It has no view of whether the person typed it, pasted it, or generated it with AI in 4 seconds. That gap is structural. It is not fixable inside the ATS.
What changed in 2024-2025
Agentic AI. Not just better resumes.
AutoApply tools apply to hundreds of roles per day. AI coaches answer screening questions live. Mass-apply platforms are VC-funded, consumer-grade, and growing. The application layer is no longer human-speed.

Background checks: post-offer. Interview tools: late funnel. ATS screening: keyword filters on submitted text. No tool sits between the Apply button and the ATS to analyze how the application was created.
The Cloudflare analogy
Between the internet and the server, sits Cloudflare. Every request passes through it. Nobody questions it. It is infrastructure.
Between the apply button and the ATS: no one. That layer does not exist yet. It will. The company that builds it first owns the infrastructure.
This is not a feature gap inside an existing tool. It is a structural gap in the architecture of hiring itself.
The Product

Sentriq sits in the gap.
Three signals. Zero friction for real candidates.

1
Behavioral Baseline
Passive. Invisible. Captured before the candidate knows they are being assessed. Typing dynamics, paste events, field focus patterns, session timing, device fingerprint, mass-apply signatures. Starts before identity exists.
Typing patterns Paste detection Session timing Device signal
2
Gap Assessor
Analyzes the submitted text for role-fit alignment, consistency between sections, experience-claim plausibility, and AI-generation patterns. Re-runs throughout the pipeline as new information arrives. Gives recruiters a structured signal, not a score.
Role-fit signals Consistency AI markers
3
Clarification Step
Triggered selectively when Layer 1 and 2 signals are inconclusive. A short async task framed as part of the application. Verifies live authorship without biometrics. Real candidates pass without friction. Zero automated rejections.
Async No biometrics Human decides
What the recruiter sees
A structured application signal profile enriching each submission inside their existing ATS. No new workflow. No new screen. The signal appears where decisions already happen.
What the candidate experiences
Nothing, for real candidates. A short clarification step only if signals are low. Never described as security. Never a CAPTCHA. A "short role-relevance check" that takes 4 minutes.

15 companies on the graph is the threshold where cross-company signal starts outperforming single-company heuristics. After that, every new node makes the detection surface larger and more precise. Competitors start from zero. We compound.

Workflow Lock-in

Resumes are text. Answers are text. Behavioral logs are structured data. This is not a computer vision problem or a video analysis problem. Text analysis at inference is near-zero cost. The economics improve as volume scales. Gross margin is structural, not optimized.
