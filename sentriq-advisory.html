The Product

Three steps. Recruiters start every review with a signal profile.

Step 1
Application Behavior Analysis

A lightweight SDK captures how the application was completed: typing rhythm, hesitation, browser behavior, session consistency. Passive and invisible. The candidate does not know they are being measured.

Typing patterns Session behavior Claim consistency
Step 2
Signal Profile Generation

A Candidate Signal Profile is generated with a confidence label. When gaps exist, a short clarification step is triggered automatically. The deterministic logic decides what to challenge. The LLM never decides what to flag.

Confidence label Gap analysis Clarification step
Step 3
ATS Enrichment

The enriched candidate profile, confidence label, explanation bullets, and recommendation route directly into the recruiter's existing ATS via webhook. The recruiter opens a queue that already has signal. Zero workflow change.

ATS-native delivery Auditable output

The question Sentriq answers for every application: how much confidence should we have that this application is real, relevant, consistent, and worth recruiter time?
