How It Works

Three layers. Zero friction for real candidates. No automated rejections.

Layer 1
Behavioral Baseline

Captures metadata as the form is filled: typing rhythm, field timing, copy-paste patterns, session consistency. Invisible to the applicant. Always active. No added friction.

Layer 2
Async Challenge

Triggered only when Layer 1 flags an anomaly. A short, role-specific task embedded in the application flow. Framed as part of the process. Surfaces proxy impersonation and AI-assisted credential fabrication with no added friction for genuine applicants.

Layer 3
Human Review

Sentriq analyst reviews the full signal stack when Layer 2 flags divergence. The employer decides. Zero automated rejections. EU AI Act compliant by design: human always in the loop.

Think of it as a CAPTCHA for hiring: invisible when everything is fine, impossible to fake when it is not. The same principle Cloudflare uses at the network layer, applied upstream of every ATS.

Structural Moat

Every employer that joins makes the signal denser for everyone.

01
Cross-Company Graph

A coordinated fraud campaign targeting 30 employers is invisible if each employer only sees their own applications. On Sentriq's network, the same attack signature surfaces across all 30 simultaneously.

02
Architecturally Unavailable to Incumbents

ATS customer contracts prohibit cross-employer data sharing. This is not a policy decision any ATS can reverse. The cross-company graph requires neutral ground. Only a platform-agnostic layer can build it.

03
Behavioral Model Compounds

Each application reviewed adds signal density to the behavioral baseline model. Zero-day fraud patterns detected at Company A are surfaced at Company B before the same attacker submits there.

The graph is the moat. Not features. Not brand. The cross-company signal network requires direct candidate consent and neutral infrastructure: structurally unavailable to every ATS, background check company, and employer-deployed product on the market.
